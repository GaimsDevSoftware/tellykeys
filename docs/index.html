What is TellyKeys?

Linux has plenty of command-line tools for Android TV. TellyKeys is the polished desktop version — approachable, visual, and comfortable for daily use.

It speaks the Android TV Remote Protocol v2 over your local network — the same family of protocol used by the official Google TV mobile app. Discover the TV, pair once with the six-character code shown on screen, then control it with large, simple buttons. ADB is optional and only used as a fallback for some text input and system shortcuts.

cd tellykeys
python3 -m venv --system-site-packages .venv
. .venv/bin/activate
python -m pip install -e .

Text input for Google TV & YouTube

Typing on Google TV is hard because every app behaves differently. TellyKeys tries several strategies so text actually lands.

YouTube

Opens a YouTube search URL directly when YouTube is the active app.

ADB fallback

Optional fallback when developer / network debugging is enabled on the TV.

Remote IME

Uses the Google TV remote text protocol as a final fallback.

If text does not appear, open Options > Text > Text input help inside the app.

Voice search

Experimental, but working on the protocol side.

TellyKeys opens a Google TV voice session and streams microphone audio as 16-bit PCM, mono, 8000 Hz. On PipeWire systems it prefers pw-record, falling back to parec. If the TV opens voice search but does not hear you, open Options > Text, select the real microphone source (not a monitor source), press Test mic, then try again. Voice search auto-stops after 3 seconds of silence following speech, or 8 seconds if no speech is heard.

FAQ

Common questions about running a Google TV remote on Linux.

Does TellyKeys work as a Google TV remote on Linux?

Yes. It controls Google TV and Android TV over your local network using the Android TV Remote Protocol v2 — the same family of protocol used by the official Google TV mobile app. No ADB is required for normal pairing and remote control.

Which Linux distributions are supported?

Linux Mint and Ubuntu are the primary targets, with a .deb package available. Fedora and other RPM distributions work from source, with RPM and Flatpak packages being added. macOS and Windows ports are planned later.

Can I send text to Google TV from Linux?

Yes. Because Google TV apps handle text differently, TellyKeys uses several strategies: a direct YouTube search when YouTube is active, an optional ADB fallback, and the remote IME protocol.

Do I need ADB or developer mode?

No. ADB is optional and only used as a fallback for some text input and system settings shortcuts. Normal pairing and remote control work without it.

Is voice search reliable?

It is experimental. It depends on your TV supporting third-party remote voice sessions, and on selecting a real microphone source in the app's settings.

Known limitations

Honest about the rough edges — TellyKeys is under active development.

  • Voice search depends on TV support for third-party remote voice sessions.
  • App-specific text input varies by Google TV app.
  • RPM and Flatpak packages are still being built; install from source on those platforms for now.
